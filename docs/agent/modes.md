# 模式

Antigravity 的代理在不同階段會進入不同的工作模式，每種模式有其特定目的。

## 三種核心模式

### 🎯 規劃模式 (Planning)

代理研究程式碼、理解需求、設計解決方案。

**特徵**：
- 閱讀檔案與程式碼庫
- 分析專案結構
- 產生 `implementation_plan.md`
- 等待用戶審核

**適用時機**：
- 開始新任務
- 遇到複雜需求
- 需要重新評估方向

### ⚡ 執行模式 (Execution)

代理實作計畫中的變更。

**特徵**：
- 編輯檔案
- 執行命令
- 操作瀏覽器
- 更新 `task.md` 進度

**適用時機**：
- 規劃獲得批准後
- 實作具體功能

### ✅ 驗證模式 (Verification)

代理測試變更、驗證結果。

**特徵**：
- 執行測試套件
- 擷取截圖
- 錄製操作過程
- 產生 `walkthrough.md`

**適用時機**：
- 執行完成後
- 需要確認結果

## 模式流程

```
      ┌──────────────┐
      │    規劃      │
      │  Planning    │
      └──────┬───────┘
             │ 批准
             ▼
      ┌──────────────┐
      │    執行      │◄────┐
      │  Execution   │     │
      └──────┬───────┘     │
             │             │ 發現問題
             ▼             │
      ┌──────────────┐     │
      │    驗證      │─────┘
      │ Verification │
      └──────────────┘
```

## 任務邊界

「任務邊界」(Task Boundary) 是模式之間的分隔點，代理會在此：

- 更新任務狀態
- 產生摘要報告
- 與用戶溝通

---

> **提示**：您可以隨時要求代理切換模式，例如「回到規劃階段」。
